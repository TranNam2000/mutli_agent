# Criteria: Business Analyst (BA)
PASS_THRESHOLD: 7
WEIGHTS: completeness=0.45 format=0.15 quality=0.40

## Completeness (what must exist)
- [ ] States clearly WHAT to build (not HOW)
- [ ] Measurable "done" conditions (not "works well")
- [ ] Large scope (>1 screen): feature list + risks + users
- [ ] Small scope (1 screen): acceptance criteria in GIVEN/WHEN/THEN (or equivalent)
- [ ] MISSING_INFO items listed if anything is unclear — or none at all

## Format (structure)
- [ ] Plain language, no jargon (PRD, MoSCoW, KPI)
- [ ] Single screen ≤ 1 page; large feature ≤ 2 pages
- [ ] Clear hierarchy (heading → bullet, not flat text)

## Quality (depth / usefulness)
- [ ] Acceptance criteria specific enough that Dev knows when "done"
- [ ] No hidden assumptions ("user will just know", "as usual")
- [ ] Risks (if any): include mitigation, not just a list
- [ ] No vague phrasing ("works normally", "as required")
